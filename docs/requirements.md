# 要件定義書

## 1. 概要

X(旧Twitter)のような、テキストベースの短文投稿を中心としたコミュニケーションツール(SNS)を開発する。
社内SNSのように、単一の組織に所属する複数のユーザーが日常的に利用することを想定する。

## 2. 目的

- 組織内のメンバー同士が気軽に情報発信・交流できる場を提供する
- 投稿に対するコメント・いいねを通じて、発信者以外(第三者)も含めたコミュニケーションを活性化する

## 3. 想定利用者

- 単一組織(会社)に所属する全ユーザー
- 特別な管理者ロールは初期スコープでは設けず、全ユーザーが同等の権限を持つ

## 4. 前提・制約

| 項目 | 内容 |
|---|---|
| 組織モデル | 単一組織前提。複数企業・マルチテナントは対象外 |
| タイムライン方式 | 「全体」(組織全体の投稿)と「フォロー中」(フォローしたユーザーの投稿のみ)をトグルで切り替え可能 |
| コメント形式 | 投稿へのコメント、およびコメントへの返信(ネスト)を許可するスレッド形式 |
| いいね対象 | 投稿に対してのみ(コメントへのいいねは対象外) |
| 第三者の関与 | 投稿者本人以外(第三者)も自由にコメント・いいねが可能。そのため投稿ごとにコメント数・いいね数を可視化する |
| フォロー導線 | ユーザー名だけでなく、投稿・コメントの投稿者表示からも直接フォロー操作が行える |
| 編集・削除 | 自分の投稿・コメントは編集・削除ができる(論理削除) |

## 5. 機能スコープ(初期リリース)

詳細は [機能一覧](features.md) を参照。

1. ログイン機能
2. タイムライン機能(全体/フォロー中の切替)
3. コメント機能(ネスト返信含む)
4. いいね機能
5. 画像投稿機能
6. フォロー機能(ユーザー名・投稿・コメントからのフォロー)
7. 投稿・コメントの編集・削除

## 6. スコープ外(初期リリースでは対応しない)

- マルチテナント(複数組織)対応
- 管理者/一般ユーザーなどのロール・権限管理
- ダイレクトメッセージ(DM)機能
- 通知機能(プッシュ通知・メール通知)
- 投稿の検索・ハッシュタグ機能
- コメントへのいいね
- 部署・役職などの組織属性によるユーザーの絞り込み

## 7. 非機能要件(初期方針)

| 分類 | 方針 |
|---|---|
| 認証 | ID(メールアドレス)・パスワードによるログイン。パスワードはハッシュ化して保存 |
| データ整合性 | いいねは1ユーザー1投稿につき1回まで、フォローは1ユーザーにつき1回まで(重複防止) |
| 画像 | 投稿1件につき複数枚の画像添付を許容 |
| 削除 | 投稿・コメントの論理削除を想定(物理削除は行わない方針を検討) |

## 8. 用語定義

| 用語 | 説明 |
|---|---|
| 投稿(Post) | ユーザーが作成するテキスト+画像のコンテンツ |
| タイムライン | フォローしているユーザーの投稿が時系列に並ぶ一覧 |
| フォロー | あるユーザーが別のユーザーの投稿を自分のタイムラインに表示させるための関係 |
| コメント | 投稿に対する返信。コメントへの返信(ネスト)も可能 |
| いいね | 投稿に対する好意的なリアクション |
